## 1. Objective

Transfer all real data, information, and services from the existing live website (**https://rallysequities.com/**) into the new layout generated by Claude (**https://claude.ai/public/artifacts/b0943060-ef8e-4954-b099-d63f5db9c7c5**), following that layout strictly.

## 2. How the layout was obtained

The Claude artifact page is JavaScript-rendered, so the full 147 KB HTML source was extracted by downloading the page and decoding the embedded Next.js flight-data payload. The decoded single-file site (HTML + CSS + JS) was saved as `artifact_reference.html` and used as the exact base for the rebuild.

## 4e. Phase 5 — Backend

The forms are no longer demos. A lightweight backend now powers the site:

**Stack:** Node.js (>= 22.5) + Express 4 + Multer 2 (uploads) + built-in `node:sqlite` (zero native dependencies) + Nodemailer 8 (optional email notifications). 0 npm audit vulnerabilities.

**Files:** [server.js](server.js) (the whole backend, ~230 lines), [admin.html](admin.html) (admin UI), [package.json](package.json), [.env.example](.env.example), `start-server.cmd` (Windows launcher).

**Endpoints:**
| Method/Path | Purpose |
|---|---|
| `POST /api/contact` | Contact form — validates name/email/subject/message against the live site's subject list |
| `POST /api/applications` | Account opening — all 5-step form fields + up to 4 documents (CNIC front/back, address proof, photo; JPG/PNG/PDF, 5 MB cap); returns a server-issued `RE-YYYY-NNNNN` reference |
| `GET /api/admin/contacts` · `/api/admin/applications` · `/api/admin/file/:ref/:name` | Token-protected admin reads (Bearer `ADMIN_TOKEN`) |
| `GET /admin` | Dark-themed admin dashboard listing applications (with document download) and messages |
| `GET /*` | Serves the static site |

**Storage:** SQLite database at `data/rallys.db` (`contacts`, `applications` tables); uploaded documents under `data/uploads/<reference>/`. The `data/` folder is git-ignored.

**Security/robustness:** input validation + length caps, MIME-type and size limits on uploads, path-traversal guard on file downloads, per-IP rate limiting (30 req / 10 min on `/api/*`), token-gated admin, HTML-escaped admin rendering. Optional SMTP notification per submission via `.env` (see `.env.example`).

**Frontend wiring:** `submitContact()` and `submitApp()` now POST to the API (the application sends `FormData` including the uploaded files and shows the server's reference number on the success screen). If no backend is reachable — e.g. the site is deployed purely statically — both forms gracefully fall back to the previous demo behavior.

**Run:**
```bash
npm install
npm start          # http://localhost:5174  (admin token printed in console)
```

**Verified end-to-end:** contact + application submitted through the real UI in a browser → rows in SQLite, CNIC file on disk, reference number displayed; bad email rejected with HTTP 400; wrong admin token rejected with 401.

## 4h. Phase 8 — Immersive experience layer

Client wanted the whole site to feel like an immersive, modern experience. A self-contained "IMMERSIVE ENGINE" (one gated IIFE + a CSS block) was added — no existing markup, data, routing, calculators, or market engine were changed:

- **Branded preloader** — a glowing logo, gold-shimmer wordmark, and animated progress bar cover the first paint, then fade out on `window.load` (min 650 ms for elegance, hard cap 2.4 s so it can never get stuck).
- **Hero particle constellation** — a canvas of drifting emerald/gold nodes that link into a living network behind the headline. Capped node count (24–54 by width), pauses when the tab is hidden or the hero scrolls out of view, and is fully disabled on mobile (<760px) and under reduced-motion.
- **Cursor spotlight** — a soft gold/emerald glow follows the cursor (screen blend, eased) — desktop fine-pointer only.
- **3D tilt cards** — service cards and PSX performer chips tilt toward the cursor in perspective (max 7°) while keeping their glass blur; reverts smoothly on leave. Desktop only.
- **Scroll progress bar** (gold gradient, top) + **back-to-top** button (glass, appears past 600px, stacked above the WhatsApp FAB).
- **Hero parallax** — the hero photo drifts at 0.14× scroll for depth.
- **SPA page transitions** — every `showPage()` navigation now fades/slides the new page in (`pageIn` 0.55 s) instead of an instant swap.

All effects are gated behind `matchMedia('(pointer:fine)')`, a width check, and `prefers-reduced-motion` (which disables the lot while keeping the preloader's dismissal and all content fully visible). Verified in headless Chrome — desktop: preloader lifecycle, particles painting, cursor glow, tilt (backdrop-blur intact), scroll bar, page transition all confirmed; mobile + reduced-motion: particles and cursor correctly off; **zero console/page errors in every mode**.
